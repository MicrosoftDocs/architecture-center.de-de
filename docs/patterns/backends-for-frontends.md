# <a name="backends-for-frontends-pattern"></a><span data-ttu-id="e6c26-104">Muster „Back-Ends für Front-Ends“</span><span class="sxs-lookup"><span data-stu-id="e6c26-104">Backends for Frontends pattern</span></span>

<span data-ttu-id="e6c26-105">Erstellen Sie separate Back-End-Dienste zur Nutzung durch bestimmte Front-End-Anwendungen oder -Schnittstellen.</span><span class="sxs-lookup"><span data-stu-id="e6c26-105">Create separate backend services to be consumed by specific frontend applications or interfaces.</span></span> <span data-ttu-id="e6c26-106">Dieses Muster ist hilfreich, wenn Sie vermeiden möchten, dass ein einzelnes Back-End für mehrere Schnittstellen angepasst werden muss.</span><span class="sxs-lookup"><span data-stu-id="e6c26-106">This pattern is useful when you want to avoid customizing a single backend for multiple interfaces.</span></span> <span data-ttu-id="e6c26-107">Dieses Muster wurde von Sam Newman erstmals beschrieben.</span><span class="sxs-lookup"><span data-stu-id="e6c26-107">This pattern was first described by Sam Newman.</span></span>

## <a name="context-and-problem"></a><span data-ttu-id="e6c26-108">Kontext und Problem</span><span class="sxs-lookup"><span data-stu-id="e6c26-108">Context and problem</span></span>

<span data-ttu-id="e6c26-109">Eine Anwendung kann ursprünglich auf eine Desktop-Webbenutzeroberfläche ausgerichtet sein.</span><span class="sxs-lookup"><span data-stu-id="e6c26-109">An application may initially be targeted at a desktop web UI.</span></span> <span data-ttu-id="e6c26-110">Normalerweise wird ein Back-End-Dienst parallel entwickelt, mit dem die für die Benutzeroberfläche benötigten Features bereitgestellt werden.</span><span class="sxs-lookup"><span data-stu-id="e6c26-110">Typically, a backend service is developed in parallel that provides the features needed for that UI.</span></span> <span data-ttu-id="e6c26-111">Wenn die Benutzerbasis der Anwendung wächst, wird eine mobile Anwendung entwickelt, die mit demselben Back-End interagieren muss.</span><span class="sxs-lookup"><span data-stu-id="e6c26-111">As the application's user base grows, a mobile application is developed that must interact with the same backend.</span></span> <span data-ttu-id="e6c26-112">Der Back-End-Dienst wird zu einem Back-End für allgemeine Zwecke, mit dem die Anforderungen der Desktopbenutzeroberfläche und mobilen Benutzeroberfläche erfüllt werden.</span><span class="sxs-lookup"><span data-stu-id="e6c26-112">The backend service becomes a general-purpose backend, serving the requirements of both the desktop and mobile interfaces.</span></span>

<span data-ttu-id="e6c26-113">Die Funktionen eines mobilen Geräts unterscheiden sich aufgrund von Bildschirmgröße, Leistung und Anzeigebeschränkungen aber erheblich von einem Desktopbrowser.</span><span class="sxs-lookup"><span data-stu-id="e6c26-113">But the capabilities of a mobile device differ significantly from a desktop browser, in terms of screen size, performance, and display limitations.</span></span> <span data-ttu-id="e6c26-114">Die Anforderungen für das Back-End einer mobilen Anwendung unterscheiden sich von der Desktop-Webbenutzeroberfläche.</span><span class="sxs-lookup"><span data-stu-id="e6c26-114">As a result, the requirements for a mobile application backend differ from the desktop web UI.</span></span>

<span data-ttu-id="e6c26-115">Diese Unterschiede führen zu konkurrierenden Anforderungen für das Back-End.</span><span class="sxs-lookup"><span data-stu-id="e6c26-115">These differences result in competing requirements for the backend.</span></span> <span data-ttu-id="e6c26-116">Für das Back-End sind regelmäßige und signifikante Änderungen erforderlich, damit es sowohl für die Desktop-Webbenutzeroberfläche als auch für die mobile Anwendung genutzt werden kann.</span><span class="sxs-lookup"><span data-stu-id="e6c26-116">The backend requires regular and significant changes to serve both the desktop web UI and the mobile application.</span></span> <span data-ttu-id="e6c26-117">Häufig arbeiten separate Benutzeroberflächenteams an den einzelnen Front-Ends, sodass das Back-End im Entwicklungsprozess zu einem Engpass wird.</span><span class="sxs-lookup"><span data-stu-id="e6c26-117">Often, separate interface teams work on each frontend, causing the backend to become a bottleneck in the development process.</span></span> <span data-ttu-id="e6c26-118">In Konflikt stehende Updateanforderungen und die Notwendigkeit, dass der Dienst für beide Front-Ends betriebsbereit gehalten werden muss, kann dazu führen, dass für eine einzelne bereitstellbare Ressource hoher Aufwand entsteht.</span><span class="sxs-lookup"><span data-stu-id="e6c26-118">Conflicting update requirements, and the need to keep the service working for both frontends, can result in spending a lot of effort on a single deployable resource.</span></span>

![Kontext- und Problemdiagramm des Musters „Back-Ends für Front-Ends“](./_images/backend-for-frontend.png)

<span data-ttu-id="e6c26-120">Da die Entwicklungsaktivitäten auf den Back-End-Dienst ausgerichtet sind, kann ggf. ein separates Team zusammengestellt werden, um das Back-End zu verwalten und zu warten.</span><span class="sxs-lookup"><span data-stu-id="e6c26-120">As the development activity focuses on the backend service, a separate team may be created to manage and maintain the backend.</span></span> <span data-ttu-id="e6c26-121">Dies führt letztendlich zu einer Trennung zwischen den Teams für die Entwicklung der Benutzeroberfläche und des Back-Ends, und das Back-End-Team muss die konkurrierenden Anforderungen der verschiedenen Benutzeroberflächenteams ausgleichen.</span><span class="sxs-lookup"><span data-stu-id="e6c26-121">Ultimately, this results in a disconnect between the interface and backend development teams, placing a burden on the backend team to balance the competing requirements of the different UI teams.</span></span> <span data-ttu-id="e6c26-122">Wenn ein Benutzeroberflächenteam Änderungen des Back-Ends fordert, müssen diese Änderungen von anderen Benutzeroberflächenteams überprüft werden, bevor sie in das Back-End integriert werden können.</span><span class="sxs-lookup"><span data-stu-id="e6c26-122">When one interface team requires changes to the backend, those changes must be validated with other interface teams before they can be integrated into the backend.</span></span>

## <a name="solution"></a><span data-ttu-id="e6c26-123">Lösung</span><span class="sxs-lookup"><span data-stu-id="e6c26-123">Solution</span></span>

<span data-ttu-id="e6c26-124">Erstellen Sie ein Back-End pro Benutzeroberfläche.</span><span class="sxs-lookup"><span data-stu-id="e6c26-124">Create one backend per user interface.</span></span> <span data-ttu-id="e6c26-125">Optimieren Sie das Verhalten und die Leistung jedes Back-Ends, um die Anforderungen der Front-End-Umgebung bestmöglich zu erfüllen, ohne sich Gedanken darüber machen zu müssen, dass andere Front-End-Umgebungen betroffen sind.</span><span class="sxs-lookup"><span data-stu-id="e6c26-125">Fine tune the behavior and performance of each backend to best match the needs of the frontend environment, without worrying about affecting other frontend experiences.</span></span>

![Diagramm des Musters „Back-Ends für Front-Ends“](./_images/backend-for-frontend-example.png)

<span data-ttu-id="e6c26-127">Da jedes Back-End spezifisch für eine Benutzeroberfläche gilt, kann es für diese Schnittstelle optimiert werden.</span><span class="sxs-lookup"><span data-stu-id="e6c26-127">Because each backend is specific to one interface, it can be optimized for that interface.</span></span> <span data-ttu-id="e6c26-128">Das Ergebnis ist, dass es kleiner, weniger komplex und meist auch schneller als ein generisches Back-End ist, mit dem versucht wird, die Anforderungen aller Benutzeroberflächen zu erfüllen.</span><span class="sxs-lookup"><span data-stu-id="e6c26-128">As a result, it will be smaller, less complex, and likely faster than a generic backend that tries to satisfy the requirements for all interfaces.</span></span> <span data-ttu-id="e6c26-129">Jedes Benutzeroberflächenteam kann das eigene Back-End selbst steuern und ist nicht von einem zentralen Back-End-Entwicklungsteam abhängig.</span><span class="sxs-lookup"><span data-stu-id="e6c26-129">Each interface team has autonomy to control their own backend and doesn't rely on a centralized backend development team.</span></span> <span data-ttu-id="e6c26-130">So kann das Benutzeroberflächenteam in den Bereichen Sprachwahl, Release-Rhythmus, Priorisierung der Workload und Featureintegration auf seinem Back-End jeweils flexibel vorgehen.</span><span class="sxs-lookup"><span data-stu-id="e6c26-130">This gives the interface team flexibility in language selection, release cadence, prioritization of workload, and feature integration in their backend.</span></span>

<span data-ttu-id="e6c26-131">Weitere Informationen finden Sie unter [Muster: Back-Ends für Front-Ends](https://samnewman.io/patterns/architectural/bff/).</span><span class="sxs-lookup"><span data-stu-id="e6c26-131">For more information, see [Pattern: Backends For Frontends](https://samnewman.io/patterns/architectural/bff/).</span></span>

## <a name="issues-and-considerations"></a><span data-ttu-id="e6c26-132">Probleme und Überlegungen</span><span class="sxs-lookup"><span data-stu-id="e6c26-132">Issues and considerations</span></span>

- <span data-ttu-id="e6c26-133">Berücksichtigen Sie, wie viele Back-Ends bereitgestellt werden sollen.</span><span class="sxs-lookup"><span data-stu-id="e6c26-133">Consider how many backends to deploy.</span></span>
- <span data-ttu-id="e6c26-134">Wenn verschiedene Oberflächen (z.B. mobile Clients) die gleichen Anforderungen senden, sollten Sie prüfen, ob die Implementierung eines Back-Ends für jede Oberfläche erforderlich ist oder ob ein einzelnes Back-End ausreicht.</span><span class="sxs-lookup"><span data-stu-id="e6c26-134">If different interfaces (such as mobile clients) will make the same requests, consider whether it is necessary to implement a backend for each interface, or if a single backend will suffice.</span></span>
- <span data-ttu-id="e6c26-135">Die Codeduplizierung über Dienste hinweg ist sehr wahrscheinlich, wenn dieses Muster implementiert wird.</span><span class="sxs-lookup"><span data-stu-id="e6c26-135">Code duplication across services is highly likely when implementing this pattern.</span></span>
- <span data-ttu-id="e6c26-136">Auf das Front-End ausgerichtete Back-End-Dienste sollten nur clientspezifische Logik und das entsprechende Verhalten aufweisen.</span><span class="sxs-lookup"><span data-stu-id="e6c26-136">Frontend-focused backend services should only contain client-specific logic and behavior.</span></span> <span data-ttu-id="e6c26-137">Die allgemeine Geschäftslogik und andere globale Features sollten an einem anderen Ort Ihrer Anwendung verwaltet werden.</span><span class="sxs-lookup"><span data-stu-id="e6c26-137">General business logic and other global features should be managed elsewhere in your application.</span></span>
- <span data-ttu-id="e6c26-138">Überlegen Sie, wie dieses Muster in Bezug auf die Zuständigkeiten eines Entwicklungsteams widergespiegelt wird.</span><span class="sxs-lookup"><span data-stu-id="e6c26-138">Think about how this pattern might be reflected in the responsibilities of a development team.</span></span>
- <span data-ttu-id="e6c26-139">Machen Sie sich Gedanken darüber, wie lange es dauert, dieses Muster zu implementieren.</span><span class="sxs-lookup"><span data-stu-id="e6c26-139">Consider how long it will take to implement this pattern.</span></span> <span data-ttu-id="e6c26-140">Führt der Aufwand für die Erstellung der neuen Back-Ends zu „technischen Schulden“, während Sie weiterhin das vorhandene generische Back-End unterstützen?</span><span class="sxs-lookup"><span data-stu-id="e6c26-140">Will the effort of building the new backends incur technical debt, while you continue to support the existing generic backend?</span></span>

## <a name="when-to-use-this-pattern"></a><span data-ttu-id="e6c26-141">Verwendung dieses Musters</span><span class="sxs-lookup"><span data-stu-id="e6c26-141">When to use this pattern</span></span>

<span data-ttu-id="e6c26-142">Verwenden Sie dieses Muster in folgenden Fällen:</span><span class="sxs-lookup"><span data-stu-id="e6c26-142">Use this pattern when:</span></span>

- <span data-ttu-id="e6c26-143">Ein freigegebener bzw. universeller Back-End-Dienst muss gewartet werden, und hierfür fällt ein erheblicher Mehraufwand bei der Entwicklung an.</span><span class="sxs-lookup"><span data-stu-id="e6c26-143">A shared or general purpose backend service must be maintained with significant development overhead.</span></span>
- <span data-ttu-id="e6c26-144">Sie möchten das Back-End für die Anforderungen bestimmter Clientoberfläche optimieren.</span><span class="sxs-lookup"><span data-stu-id="e6c26-144">You want to optimize the backend for the requirements of specific client interfaces.</span></span>
- <span data-ttu-id="e6c26-145">Anpassungen werden für ein Allzweck-Back-End vorgenommen, um mehrere Oberflächen abzudecken.</span><span class="sxs-lookup"><span data-stu-id="e6c26-145">Customizations are made to a general-purpose backend to accommodate multiple interfaces.</span></span>
- <span data-ttu-id="e6c26-146">Eine Alternativsprache ist für das Back-End einer anderen Benutzeroberfläche besser geeignet.</span><span class="sxs-lookup"><span data-stu-id="e6c26-146">An alternative language is better suited for the backend of a different user interface.</span></span>

- <span data-ttu-id="e6c26-149">Wenn nur eine Benutzeroberfläche für die Interaktion mit dem Back-End verwendet wird.</span><span class="sxs-lookup"><span data-stu-id="e6c26-149">When only one interface is used to interact with the backend.</span></span>
