# <a name="sidecar-pattern"></a><span data-ttu-id="9fe26-103">Sidecar-Muster</span><span class="sxs-lookup"><span data-stu-id="9fe26-103">Sidecar pattern</span></span>

## <a name="context-and-problem"></a><span data-ttu-id="9fe26-110">Kontext und Problem</span><span class="sxs-lookup"><span data-stu-id="9fe26-110">Context and Problem</span></span>

<span data-ttu-id="9fe26-111">Anwendungen und Dienste benötigen häufig zusammengehörige Funktionen, wie z. B. Überwachung, Protokollierung, Konfiguration und Netzwerkdienste.</span><span class="sxs-lookup"><span data-stu-id="9fe26-111">Applications and services often require related functionality, such as monitoring, logging, configuration, and networking services.</span></span> <span data-ttu-id="9fe26-112">Diese Peripherieaufgaben können als separate Komponenten oder Dienste implementiert werden.</span><span class="sxs-lookup"><span data-stu-id="9fe26-112">These peripheral tasks can be implemented as separate components or services.</span></span> 

<span data-ttu-id="9fe26-113">Wenn sie eng in die Anwendung integriert sind, können sie in demselben Prozess wie die Anwendung ausgeführt werden und dadurch gemeinsam verwendete Ressourcen effizient nutzen.</span><span class="sxs-lookup"><span data-stu-id="9fe26-113">If they are tightly integrated into the application, they can run in the same process as the application, making efficient use of shared resources.</span></span> <span data-ttu-id="9fe26-114">Dies bedeutet aber auch, dass sie nicht gut isoliert sind, und ein Ausfall in einer dieser Komponenten kann Auswirkungen auf andere Komponenten oder die gesamte Anwendung haben.</span><span class="sxs-lookup"><span data-stu-id="9fe26-114">However, this also means they are not well isolated, and an outage in one of these components can affect other components or the entire application.</span></span> <span data-ttu-id="9fe26-115">Außerdem müssen sie üblicherweise mit derselben Sprache wie die übergeordnete Anwendung implementiert werden.</span><span class="sxs-lookup"><span data-stu-id="9fe26-115">Also, they usually need to be implemented using the same language as the parent application.</span></span> <span data-ttu-id="9fe26-116">Daher weisen die Komponente und die Anwendung eine starke wechselseitige Abhängigkeit auf.</span><span class="sxs-lookup"><span data-stu-id="9fe26-116">As a result, the component and the application have close interdependence on each other.</span></span>

<span data-ttu-id="9fe26-117">Wenn die Anwendung in Dienste aufgegliedert wird, kann jeder Dienst mit einer anderen Sprache und anderer Technologie erstellt werden.</span><span class="sxs-lookup"><span data-stu-id="9fe26-117">If the application is decomposed into services, then each service can be built using different languages and technologies.</span></span> <span data-ttu-id="9fe26-118">Zwar bietet dies mehr Flexibilität, doch bedeutet es auch, dass jede Komponente eigene Abhängigkeiten aufweist und sprachspezifische Bibliotheken für den Zugriff auf die zugrunde liegende Plattform und alle mit der übergeordneten Anwendung gemeinsam verwendeten Ressourcen benötigt.</span><span class="sxs-lookup"><span data-stu-id="9fe26-118">While this gives more flexibility, it means that each component has its own dependencies and requires language-specific libraries to access the underlying platform and any resources shared with the parent application.</span></span> <span data-ttu-id="9fe26-119">Darüber hinaus kann eine Bereitstellung dieser Funktionen als separate Dienste zu höherer Latenz der Anwendung führen.</span><span class="sxs-lookup"><span data-stu-id="9fe26-119">In addition, deploying these features as separate services can add latency to the application.</span></span> <span data-ttu-id="9fe26-120">Das Verwalten des Codes und der Abhängigkeiten für diese sprachspezifischen Schnittstellen kann auch zu wesentlich höherer Komplexität führen, insbesondere bei Hosting, Bereitstellung und Verwaltung.</span><span class="sxs-lookup"><span data-stu-id="9fe26-120">Managing the code and dependencies for these language-specific interfaces can also add considerable complexity, especially for hosting, deployment, and management.</span></span>

## <a name="solution"></a><span data-ttu-id="9fe26-121">Lösung</span><span class="sxs-lookup"><span data-stu-id="9fe26-121">Solution</span></span>

<span data-ttu-id="9fe26-122">Verbinden Sie eine zusammenhängende Gruppe von Aufgaben mit der primären Anwendung, aber platzieren Sie diese in einem eigenen Prozess oder Container, um eine homogene Schnittstelle für Plattformdienste sprachübergreifend bereitzustellen.</span><span class="sxs-lookup"><span data-stu-id="9fe26-122">Co-locate a cohesive set of tasks with the primary application, but place them inside their own process or container, providing a homogeneous interface for platform services across languages.</span></span> 

![](./_images/sidecar.png)

<span data-ttu-id="9fe26-123">Ein Sidecar-Dienst ist nicht notwendigerweise Teil der Anwendung, ist aber mit ihr verbunden.</span><span class="sxs-lookup"><span data-stu-id="9fe26-123">A sidecar service is not necessarily part of the application, but is connected to it.</span></span> <span data-ttu-id="9fe26-124">Er befindet sich dort, wo sich auch die übergeordnete Anwendung befindet.</span><span class="sxs-lookup"><span data-stu-id="9fe26-124">It goes wherever the parent application goes.</span></span> <span data-ttu-id="9fe26-125">Sidecars sind unterstützte Prozesse oder Dienste, die mit der primären Anwendung bereitgestellt werden.</span><span class="sxs-lookup"><span data-stu-id="9fe26-125">Sidecars are supporting processes or services that are deployed with the primary application.</span></span> <span data-ttu-id="9fe26-126">Bei einem Motorrad wird ein Sidecar (Beiwagen) an ein einzelnes Fahrzeug angebracht, und jedes Motorrad kann einen eigenen Beiwagen haben.</span><span class="sxs-lookup"><span data-stu-id="9fe26-126">On a motorcycle, the sidecar is attached to one motorcycle, and each motorcycle can have its own sidecar.</span></span> <span data-ttu-id="9fe26-127">Auf die gleiche Weise teilt auch ein Sidecar-Dienst das Schicksal der übergeordneten Anwendung.</span><span class="sxs-lookup"><span data-stu-id="9fe26-127">In the same way, a sidecar service shares the fate of its parent application.</span></span> <span data-ttu-id="9fe26-128">Für jede Instanz der Anwendung wird eine Instanz des Sidecar bereitgestellt und parallel gehostet.</span><span class="sxs-lookup"><span data-stu-id="9fe26-128">For each instance of the application, an instance of the sidecar is deployed and hosted alongside it.</span></span> 

<span data-ttu-id="9fe26-129">Die Verwendung eines Sidecar-Musters bietet unter anderem folgende Vorteile:</span><span class="sxs-lookup"><span data-stu-id="9fe26-129">Advantages of using a sidecar pattern include:</span></span>

- <span data-ttu-id="9fe26-130">Ein Sidecar ist in Hinsicht auf die Laufzeitumgebung und die Programmiersprache von der primären Anwendung unabhängig, sodass Sie nicht einen Sidecar pro Sprache entwickeln müssen.</span><span class="sxs-lookup"><span data-stu-id="9fe26-130">A sidecar is independent from its primary application in terms of runtime environment and programming language, so you don't need to develop one sidecar per language.</span></span> 

- <span data-ttu-id="9fe26-131">Der Sidecar kann auf dieselben Ressourcen wie die primäre Anwendung zugreifen.</span><span class="sxs-lookup"><span data-stu-id="9fe26-131">The sidecar can access the same resources as the primary application.</span></span> <span data-ttu-id="9fe26-132">Beispielsweise kann ein Sidecar Systemressourcen überwachen, die sowohl vom Sidecar als auch der primären Anwendung verwendet werden.</span><span class="sxs-lookup"><span data-stu-id="9fe26-132">For example, a sidecar can monitor system resources used by both the sidecar and the primary application.</span></span> 

- <span data-ttu-id="9fe26-133">Aufgrund der Nähe zur primären Anwendung besteht keine wesentliche Latenz bei der Kommunikation zwischen beiden.</span><span class="sxs-lookup"><span data-stu-id="9fe26-133">Because of its proximity to the primary application, there’s no significant latency when communicating between them.</span></span>

<span data-ttu-id="9fe26-135">Das Sidecar-Muster wird häufig mit Containern verwendet und als Sidecar-Container oder Sidekick-Container bezeichnet.</span><span class="sxs-lookup"><span data-stu-id="9fe26-135">The sidecar pattern is often used with containers and referred to as a sidecar container or sidekick container.</span></span>
