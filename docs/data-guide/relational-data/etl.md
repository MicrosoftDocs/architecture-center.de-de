<span data-ttu-id="76c74-109">Extrahieren, Transformieren und Laden (ETL) ist eine Datenpipeline und wird verwendet, um Daten aus verschiedenen Quellen zu sammeln, gemäß den Geschäftsregeln zu transformieren und in einen Zieldatenspeicher zu laden.</span><span class="sxs-lookup"><span data-stu-id="76c74-109">Extract, transform, and load (ETL) is a data pipeline used to collect data from various sources, transform the data according to business rules, and load it into a destination data store.</span></span> <span data-ttu-id="76c74-110">Die Transformation in ETL findet in einem speziellen Modul statt und beinhaltet häufig die Verwendung von Stagingtabellen zur vorübergehenden Speicherung von Daten, während diese transformiert und schließlich in das Ziel geladen werden.</span><span class="sxs-lookup"><span data-stu-id="76c74-110">The transformation work in ETL takes place in a specialized engine, and often involves using staging tables to temporarily hold data as it is being transformed and ultimately loaded to its destination.</span></span>

<span data-ttu-id="76c74-111">Die durchgeführte Datentransformation umfasst in der Regel verschiedene Vorgänge wie Filtern, Sortieren, Aggregieren, Verknüpfen, Bereinigen, Deduplizieren und Validieren von Daten.</span><span class="sxs-lookup"><span data-stu-id="76c74-111">The data transformation that takes place usually involves various operations, such as filtering, sorting, aggregating, joining data, cleaning data, deduplicating, and validating data.</span></span>

## <a name="extract-load-and-transform-elt"></a><span data-ttu-id="76c74-119">Extrahieren, Laden und Transformieren (ELT)</span><span class="sxs-lookup"><span data-stu-id="76c74-119">Extract, load, and transform (ELT)</span></span>

<span data-ttu-id="76c74-120">Extrahieren, Laden und Transformieren (ELT) unterscheidet sich von ETL lediglich darin, an welcher Stelle die Transformation erfolgt.</span><span class="sxs-lookup"><span data-stu-id="76c74-120">Extract, load, and transform (ELT) differs from ETL solely in where the transformation takes place.</span></span> <span data-ttu-id="76c74-121">In der ELT-Pipeline findet die Transformation im Zieldatenspeicher statt.</span><span class="sxs-lookup"><span data-stu-id="76c74-121">In the ELT pipeline, the transformation occurs in the target data store.</span></span> <span data-ttu-id="76c74-122">Die Daten werden mithilfe der Verarbeitungsfunktionen des Zieldatenspeichers transformiert, anstatt ein separates Transformationsmodul zu verwenden.</span><span class="sxs-lookup"><span data-stu-id="76c74-122">Instead of using a separate transformation engine, the processing capabilities of the target data store are used to transform data.</span></span> <span data-ttu-id="76c74-123">Dadurch wird das Transformationsmodul aus der Pipeline entfernt, was die Architektur vereinfacht.</span><span class="sxs-lookup"><span data-stu-id="76c74-123">This simplifies the architecture by removing the transformation engine from the pipeline.</span></span> <span data-ttu-id="76c74-124">Ein weiterer Vorteil dieses Ansatzes ist, dass durch Skalieren des Zieldatenspeichers auch die Leistung der ELT-Pipeline skaliert wird.</span><span class="sxs-lookup"><span data-stu-id="76c74-124">Another benefit to this approach is that scaling the target data store also scales the ELT pipeline performance.</span></span> <span data-ttu-id="76c74-125">ELT setzt jedoch voraus, dass das Zielsystem über genügend Leistung verfügt, um die Daten effizient transformieren zu können.</span><span class="sxs-lookup"><span data-stu-id="76c74-125">However, ELT only works well when the target system is powerful enough to transform the data efficiently.</span></span>

<span data-ttu-id="76c74-127">ELT kommt üblicherweise in Big Data-Szenarien zum Einsatz.</span><span class="sxs-lookup"><span data-stu-id="76c74-127">Typical use cases for ELT fall within the big data as Hadoop distributed file system (HDFS) or Azure Data Lake Store.</span></span> <span data-ttu-id="76c74-129">Anschließend können Sie die Quelldaten mit Technologien wie Spark, Hive oder PolyBase abfragen.</span><span class="sxs-lookup"><span data-stu-id="76c74-129">Technologies such as Spark, Hive, or PolyBase can then be used to query the source data.</span></span> <span data-ttu-id="76c74-130">Entscheidend bei ELT ist, dass der für die Transformation verwendete Datenspeicher der gleiche Datenspeicher ist, in dem die Daten später auch genutzt werden.</span><span class="sxs-lookup"><span data-stu-id="76c74-130">The key point with ELT is that the data store used to perform the transformation is the same data store where the data is ultimately data-stu-id="76c74-132">This approach skips the data copy step present in ETL, which can be a time consuming operation for large data sets.</span></span>

<span data-ttu-id="76c74-133">In der Praxis ist der Zieldatenspeicher ein [Data Warehouse](./data-warehousing.md) mit einem Hadoop-Cluster (Hive oder Spark) oder ein SQL Data Warehouse.</span><span class="sxs-lookup"><span data-stu-id="76c74-133">In practice, the target data store is a [data warehouse](./data-warehousing.md) using either a Hadoop cluster (using Hive or Spark) or a SQL Data Warehouse.</span></span> <span data-ttu-id="76c74-134">Im Allgemeinen wird bei der Abfrage ein Schema auf die Flatfiledaten angewendet und als Tabelle gespeichert, wodurch die Daten wie jede andere Tabelle im Datenspeicher abgefragt werden können.</span><span class="sxs-lookup"><span data-stu-id="76c74-134">In general, a schema is overlaid on the flat file data at query time and stored as a table, enabling the data to be queried like any other table in the data store.</span></span> <span data-ttu-id="76c74-135">Diese Tabellen werden als externe Tabellen bezeichnet, da sich die Daten nicht in dem Speicher befinden, der vom Datenspeicher selbst verwaltet wird, sondern in einem externen skalierbaren Speicher.</span><span class="sxs-lookup"><span data-stu-id="76c74-135">These are referred to as external tables because the data does not reside in storage managed by the data store itself, but on some external scalable storage.</span></span>

<span data-ttu-id="76c74-141">In der letzten Phase der ELT-Pipeline werden die Quelldaten üblicherweise in ein endgültiges Format transformiert, das besser für die Arten von Abfragen geeignet ist, die unterstützt werden müssen.</span><span class="sxs-lookup"><span data-stu-id="76c74-141">The final phase of the ELT pipeline is typically to transform the source data into a final format that is more efficient for the types of queries that need to be supported.</span></span> <span data-ttu-id="76c74-142">So können die Daten beispielsweise partitioniert werden.</span><span class="sxs-lookup"><span data-stu-id="76c74-142">For example, the data may be partitioned.</span></span> <span data-ttu-id="76c74-143">ELT kann außerdem optimierte Speicherformate wie Parquet verwendet, das zeilenorientierte Daten spaltenförmig speichert und eine optimierte Indizierung bietet.</span><span class="sxs-lookup"><span data-stu-id="76c74-143">Also, ELT might use optimized storage formats like Parquet, which stores row-oriented data in a columnar fashion and providess optimized indexing.</span></span>
