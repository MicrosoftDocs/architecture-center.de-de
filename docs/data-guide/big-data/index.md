## <a name="lambda-architecture"></a><span data-ttu-id="c66c9-169">Lambda-Architektur</span><span class="sxs-lookup"><span data-stu-id="c66c9-169">Lambda architecture</span></span>

<span data-ttu-id="c66c9-170">Bei sehr großen Datasets kann die Ausführung der von Clients benötigten Abfragen sehr lange dauern.</span><span class="sxs-lookup"><span data-stu-id="c66c9-170">When working with very large data sets, it can take a long time to run the sort of queries that clients need.</span></span> <span data-ttu-id="c66c9-171">Diese Abfragen können nicht in Echtzeit durchgeführt werden und erfordern häufig Algorithmen wie [MapReduce](https://en.wikipedia.org/wiki/MapReduce), die parallel für das gesamte Dataset ausgeführt werden.</span><span class="sxs-lookup"><span data-stu-id="c66c9-171">These queries can't be performed in real time, and often require algorithms such as [MapReduce](https://en.wikipedia.org/wiki/MapReduce) that operate in parallel across the entire data set.</span></span> <span data-ttu-id="c66c9-172">Die Ergebnisse werden dann getrennt von den Rohdaten gespeichert und für Abfragen verwendet.</span><span class="sxs-lookup"><span data-stu-id="c66c9-172">The results are then stored separately from the raw data and used for querying.</span></span>

<span data-ttu-id="c66c9-173">Ein Nachteil dieses Ansatzes ist die damit verbundene Wartezeit: Wenn die Verarbeitung einige Stunden dauert, gibt eine Abfrage unter Umständen Ergebnisse zurück, die bereits mehrere Stunden alt sind.</span><span class="sxs-lookup"><span data-stu-id="c66c9-173">One drawback to this approach is that it introduces latency &mdash; if processing takes a few hours, a query may return results that are several hours old.</span></span> <span data-ttu-id="c66c9-174">Im Idealfall möchten Sie einige Ergebnisse in Echtzeit erhalten (ggf. auf Kosten der Genauigkeit) und sie mit den Ergebnissen aus der Batchanalyse kombinieren.</span><span class="sxs-lookup"><span data-stu-id="c66c9-174">Ideally, you would like to get some results in real time (perhaps with some loss of accuracy), and combine these results with the results from the batch analytics.</span></span>

<span data-ttu-id="c66c9-175">Die **Lambda-Architektur** wurde erstmals von Nathan Marz vorgeschlagen und löst dieses Problem durch die Erstellung zweier Datenflusspfade.</span><span class="sxs-lookup"><span data-stu-id="c66c9-175">The **lambda architecture**, first proposed by Nathan Marz, addresses this problem by creating two paths for data flow.</span></span> <span data-ttu-id="c66c9-176">Alle beim System eingehenden Daten durchlaufen die beiden folgenden Pfade:</span><span class="sxs-lookup"><span data-stu-id="c66c9-176">All data coming into the system goes through these two paths:</span></span>

- <span data-ttu-id="c66c9-177">Eine **Batchebene** (Pfad für kalte Daten) speichert alle eingehenden Daten als Rohdaten und führt eine Batchverarbeitung der Daten durch.</span><span class="sxs-lookup"><span data-stu-id="c66c9-177">A **batch layer** (cold path) stores all of the incoming data in its raw form and performs batch processing on the data.</span></span> <span data-ttu-id="c66c9-178">Das Ergebnis dieser Verarbeitung wird als **Batchansicht** gespeichert.</span><span class="sxs-lookup"><span data-stu-id="c66c9-178">The result of this processing is stored as a **batch view**.</span></span>

- <span data-ttu-id="c66c9-179">Eine **Geschwindigkeitsebene** (Pfad für heiße Daten) analysiert Daten in Echtzeit.</span><span class="sxs-lookup"><span data-stu-id="c66c9-179">A **speed layer** (hot path) analyzes data in real time.</span></span> <span data-ttu-id="c66c9-180">Die geringe Wartezeit dieser Ebene geht zulasten der Genauigkeit.</span><span class="sxs-lookup"><span data-stu-id="c66c9-180">This layer is designed for low latency, at the expense of accuracy.</span></span>

<span data-ttu-id="c66c9-181">Die Daten der Batchebene werden einer **Bereitstellungsebene** zugeführt, die die Batchansicht indiziert, um effiziente Abfragen zu ermöglichen.</span><span class="sxs-lookup"><span data-stu-id="c66c9-181">The batch layer feeds into a **serving layer** that indexes the batch view for efficient querying.</span></span> <span data-ttu-id="c66c9-182">Die Geschwindigkeitsebene aktualisiert die Bereitstellungsebene nach und nach mit den neuesten Daten.</span><span class="sxs-lookup"><span data-stu-id="c66c9-182">The speed layer updates the serving layer with incremental updates based on the most recent data.</span></span>

![Diagramm der Lambda-Architektur](./images/lambda.png)

<span data-ttu-id="c66c9-184">Daten, die den Pfad für heiße Daten durchlaufen, werden durch Wartezeitanforderungen der Geschwindigkeitsebene eingeschränkt, um eine schnellstmögliche Verarbeitung zu ermöglichen.</span><span class="sxs-lookup"><span data-stu-id="c66c9-184">Data that flows into the hot path is constrained by latency requirements imposed by the speed layer, so that it can be processed as quickly as possible.</span></span> <span data-ttu-id="c66c9-185">Hierbei müssen häufig gewisse Abstriche bei der Genauigkeit gemacht werden, um eine möglichst schnelle Verfügbarkeit der Daten zu erreichen.</span><span class="sxs-lookup"><span data-stu-id="c66c9-185">Often, this requires a tradeoff of some level of accuracy in favor of data that is ready as quickly as possible.</span></span> <span data-ttu-id="c66c9-186">Stellen Sie sich beispielsweise ein IoT-Szenario vor, in dem Telemetriedaten von einer großen Anzahl von Temperatursensoren übermittelt werden.</span><span class="sxs-lookup"><span data-stu-id="c66c9-186">For example, consider an IoT scenario where a large number of temperature sensors are sending telemetry data.</span></span> <span data-ttu-id="c66c9-187">Die Geschwindigkeitsebene kann zur Verarbeitung eines gleitenden Zeitfensters der eingehenden Daten verwendet werden.</span><span class="sxs-lookup"><span data-stu-id="c66c9-187">The speed layer may be used to process a sliding time window of the incoming data.</span></span>

<span data-ttu-id="c66c9-188">Daten, die den Pfad für kalte Daten durchlaufen, sind dagegen nicht den gleichen Anforderungen für kurze Wartezeiten unterworfen.</span><span class="sxs-lookup"><span data-stu-id="c66c9-188">Data flowing into the cold path, on the other hand, is not subject to the same low latency requirements.</span></span> <span data-ttu-id="c66c9-189">Dies ermöglicht äußerst zeitaufwendige Berechnungen mit hoher Genauigkeit für umfangreiche Datasets.</span><span class="sxs-lookup"><span data-stu-id="c66c9-189">This allows for high accuracy computation across large data sets, which can be very time intensive.</span></span>

<span data-ttu-id="c66c9-190">Die Pfade für heiße und kalte Daten werden schließlich in der Analyseclientanwendung zusammengeführt.</span><span class="sxs-lookup"><span data-stu-id="c66c9-190">Eventually, the hot and cold paths converge at the analytics client application.</span></span> <span data-ttu-id="c66c9-191">Wenn der Client zeitnahe, aber unter Umständen ungenauere Daten in Echtzeit anzeigen muss, wird das Ergebnis aus dem Pfad für heiße Daten abgerufen.</span><span class="sxs-lookup"><span data-stu-id="c66c9-191">If the client needs to display timely, yet potentially less accurate data in real time, it will acquire its result from the hot path.</span></span> <span data-ttu-id="c66c9-192">Andernfalls werden die Ergebnisse aus dem Pfad für kalte Daten verwendet, um weniger aktuelle, dafür aber genauere Daten anzuzeigen.</span><span class="sxs-lookup"><span data-stu-id="c66c9-192">Otherwise, it will select results from the cold path to display less timely but more accurate data.</span></span> <span data-ttu-id="c66c9-193">Mit anderen Worten: Der Pfad für heiße Daten liefert Daten für ein relativ kleines Zeitfenster, nach dem die Ergebnisse mit möglichst genauen Daten aus dem Pfad für kalte Daten aktualisiert werden können.</span><span class="sxs-lookup"><span data-stu-id="c66c9-193">In other words, the hot path has data for a relatively small window of time, after which the results can be updated with more accurate data from the cold path.</span></span>

<span data-ttu-id="c66c9-194">Die auf der Batchebene gespeicherten Rohdaten sind unveränderlich.</span><span class="sxs-lookup"><span data-stu-id="c66c9-194">The raw data stored at the batch layer is immutable.</span></span> <span data-ttu-id="c66c9-195">Eingehende Daten werden immer am Ende der bereits vorhandene Daten hinzugefügt, und die vorherigen Daten werden niemals überschrieben.</span><span class="sxs-lookup"><span data-stu-id="c66c9-195">Incoming data is always appended to the existing data, and the previous data is never overwritten.</span></span> <span data-ttu-id="c66c9-196">Änderungen am Wert eines bestimmten Bezugs werden als neuer Ereignisdatensatz mit Zeitstempel gespeichert.</span><span class="sxs-lookup"><span data-stu-id="c66c9-196">Any changes to the value of a particular datum are stored as a new timestamped event record.</span></span> <span data-ttu-id="c66c9-197">Dies ermöglicht die Neuberechnung zu einem beliebigen Zeitpunkt innerhalb des Verlaufs der gesammelten Daten.</span><span class="sxs-lookup"><span data-stu-id="c66c9-197">This allows for recomputation at any point in time across the history of the data collected.</span></span> <span data-ttu-id="c66c9-198">Die Möglichkeit zur Neuberechnung der Batchansicht auf der Grundlage der ursprünglichen Rohdaten ist wichtig, da es die Erstellung neuer Ansichten ermöglicht, wenn sich das System weiterentwickelt.</span><span class="sxs-lookup"><span data-stu-id="c66c9-198">The ability to recompute the batch view from the original raw data is important, because it allows for new views to be created as the system evolves.</span></span>

## <a name="kappa-architecture"></a><span data-ttu-id="c66c9-199">Kappa-Architektur</span><span class="sxs-lookup"><span data-stu-id="c66c9-199">Kappa architecture</span></span>

<span data-ttu-id="c66c9-200">Ein Nachteil der Lambda-Architektur ist ihre Komplexität.</span><span class="sxs-lookup"><span data-stu-id="c66c9-200">A drawback to the lambda architecture is its complexity.</span></span> <span data-ttu-id="c66c9-201">Die Verarbeitungslogik kommt an zwei verschiedenen Stellen zur Anwendung (am Pfad für kalte Daten und am Pfad für heiße Daten) und verwendet unterschiedliche Frameworks.</span><span class="sxs-lookup"><span data-stu-id="c66c9-201">Processing logic appears in two different places &mdash; the cold and hot paths &mdash; using different frameworks.</span></span> <span data-ttu-id="c66c9-202">Dies hat eine Duplizierung der Berechnungslogik sowie eine komplexe Verwaltung der Architektur für beide Pfade zur Folge.</span><span class="sxs-lookup"><span data-stu-id="c66c9-202">This leads to duplicate computation logic and the complexity of managing the architecture for both paths.</span></span>

<span data-ttu-id="c66c9-203">Die **Kappa-Architektur** wurde von Jay Kreps als Alternative zur Lambda-Architektur vorgeschlagen.</span><span class="sxs-lookup"><span data-stu-id="c66c9-203">The **kappa architecture** was proposed by Jay Kreps as an alternative to the lambda architecture.</span></span> <span data-ttu-id="c66c9-204">Sie weist die gleichen grundlegenden Ziele wie die Lambda-Architektur auf, jedoch mit einem wichtigen Unterschied: Alle Daten fließen über ein Datenstrom-Verarbeitungssystem durch einen einzelnen Pfad.</span><span class="sxs-lookup"><span data-stu-id="c66c9-204">It has the same basic goals as the lambda architecture, but with an important distinction: All data flows through a single path, using a stream processing system.</span></span>

<span data-ttu-id="c66c9-206">Es gibt gewisse Ähnlichkeiten mit der Batchebene der Lambda-Architektur.
