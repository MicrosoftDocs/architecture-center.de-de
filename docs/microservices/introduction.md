- <span data-ttu-id="b894f-123">**Kompakter Code, kleine Teams:**</span><span class="sxs-lookup"><span data-stu-id="b894f-123">**Small code, small teams.**</span></span> <span monolithischen Anwendung werden Codeabhängigkeiten im Laufe der Zeit häufig unübersichtlich, sodass zum Hinzufügen eines neuen Features Code an zahlreichen Stellen bearbeitet werden muss.</span><span class="sxs-lookup"><span data-stu-id="b894f-126">In a large monolithic application, there is a tendency over time for code dependencies to become tangled, so that adding a new feature requires Abhängigkeiten und vereinfacht dadurch das Hinzufügen neuer Features.</span><span class="sxs-lookup"><span data-stu-id="b894f-127">By not sharing code or data stores, a microservices architecture - <span data-ttu-id="b894f-132">**Kombination verschiedener Technologien:**</span><span class="sxs-lookup"><span data-stu-id="b894f-132">**Mix of technologies**.</span></span> <span data-ttu-id="b894f-133">Teams können sich für die Technologie entscheiden, die am besten zu ihrem Dienst passt, und eine Kombination aus geeigneten Technologiestapeln verwenden.</span><span class="sxs-lookup"><span data-stu-id="b894f-133">Teams can pick the technology that best fits their service, using a mix of technology stacks as appropriate.</span></span>

- <span data-ttu-id="b894f-136">**Skalierbarkeit**.</span><span class="sxs-lookup"><span data-stu-id="b894f-136">**Scalability**.</span></span> <span data-ttu-id="b894f-137">In einer Microservices-Architektur kann jeder Microservice unabhängig skaliert werden.</span><span class="sxs-lookup"><span data-stu-id="b894f-137">A microservices architecture allows each microservice to be scaled independently of the others.</span></span> <span data-ttu-id="b894f-138">Dadurch lassen sich Subsysteme, die mehr Ressourcen benötigen, horizontal hochskalieren, ohne die gesamte Anwendung hochzuskalieren.</span><span class="sxs-lookup"><span data-stu-id="b894f-138">That lets you scale out subsystems that require more resources, without scaling out the entire application.</span></span> <span data-ttu-id="b894f-139">Wenn Sie Dienste in Containern bereitstellen, können Sie Microservices zudem dichter auf einem einzelnen Host platzieren und Ressourcen effizienter nutzen.</span><span class="sxs-lookup"><span data-stu-id="b894f-139">If you deploy services inside containers, you can also pack a higher density of microservices onto a single host, which allows for more efficient utilization of resources.</span></span>

- <span data-ttu-id="b894f-140">**Datenisolation:**</span><span class="sxs-lookup"><span data-stu-id="b894f-140">**Data isolation**.</span></span> <span data-ttu-id="b894f-141">Schemaaktualisierungen sind wesentlich einfacher, da nur ein einzelner Microservice betroffen ist.</span><span class="sxs-lookup"><span data-stu-id="b894f-141">It is much easier to perform schema updates, because only a single microservice is impacted.</span></span> <span data-ttu-id="b894f-142">Bei einer monolithischen Anwendung können sich Schemaaktualisierungen als echte Herausforderung erweisen, da verschiedene Teile der Anwendung unter Umständen die gleichen Daten nutzen, was jegliche Anpassung des Schemas zu einer riskanten Angelegenheit macht.</span><span class="sxs-lookup"><span data-stu-id="b894f-142">In a monolithic application, schema updates can become very challenging, because different parts of the application may all touch the same data, making any alterations to the schema risky.</span></span>

<span data-ttu-id="b894f-150">Die Vorteile von Microservices gibt es jedoch nicht umsonst.</span><span class="sxs-lookup"><span data-stu-id="b894f-150">The benefits of microservices don't come for free.</span></span> <span data-ttu-id="b894f-151">Im Anschluss finden Sie einige der Herausforderungen, die Sie vor der Erstellung einer Microservices-Architektur berücksichtigen sollten.</span><span class="sxs-lookup"><span data-stu-id="b894f-151">Here are some of the challenges to consider before embarking on a microservices architecture.</span></span>
