## <a name="why-microservices"></a><span data-ttu-id="56c4a-133">Gründe für die Verwendung von Microservices</span><span class="sxs-lookup"><span data-stu-id="56c4a-133">Why Microservices?</span></span>

<span data-ttu-id="56c4a-134">Eine ausführliche Auseinandersetzung mit Mikroservices geht über den Rahmen dieses Artikels hinaus, aber hier folgen einige der Vorteile, die Tailspin durch den Umstieg auf eine Mikroservices-Architektur erzielen möchte:</span><span class="sxs-lookup"><span data-stu-id="56c4a-134">An in-depth discussion of microservices is beyond scope of this article, but here are some of the benefits that Tailspin hopes to get by moving to a microservices architecture:</span></span>

- <span data-ttu-id="56c4a-135">**Anwendungsupgrades**:</span><span class="sxs-lookup"><span data-stu-id="56c4a-135">**Application upgrades**.</span></span> <span data-ttu-id="56c4a-136">Dienste können unabhängig voneinander bereitgestellt werden, sodass Sie einen inkrementellen Ansatz zum Ausführen von Upgrades für eine Anwendung verfolgen können.</span><span class="sxs-lookup"><span data-stu-id="56c4a-136">Services can be deployed independently, so you can take an incremental approach to upgrading an application.</span></span>
- <span data-ttu-id="56c4a-137">**Resilienz und Fehlerisolierung**:</span><span class="sxs-lookup"><span data-stu-id="56c4a-137">**Resiliency and fault isolation**.</span></span> <span data-ttu-id="56c4a-138">Wenn ein Dienst fehlerhaft ist, werden andere Dienste weiterhin ausgeführt.</span><span class="sxs-lookup"><span data-stu-id="56c4a-138">If a service fails, other services continue to run.</span></span>

- <span data-ttu-id="56c4a-145">**Kleine, fokussierte Teams**:</span><span class="sxs-lookup"><span data-stu-id="56c4a-145">**Small, focused teams**.</span></span> <span data-ttu-id="56c4a-146">Da die Anwendung in viele kleine Dienste unterteilt ist, kann jeder Dienst von einem kleinen, fokussierten Team erstellt werden.</span><span class="sxs-lookup"><span data-stu-id="56c4a-146">Because the application is broken down into many small services, each service can be built by a small focused team.</span></span>

## <a name="why-service-fabric"></a><span data-ttu-id="56c4a-147">Gründe für Service Fabric</span><span class="sxs-lookup"><span data-stu-id="56c4a-147">Why Service Fabric?</span></span>
      
<span data-ttu-id="56c4a-148">Service Fabric eignet sich gut für eine Mikroservices-Architektur, da die meisten Funktionen, die in einem verteilten System benötigt werden, in Service Fabric integriert sind, einschließlich:</span><span class="sxs-lookup"><span data-stu-id="56c4a-148">Service Fabric is a good fit for a microservices architecture, because most of the features needed in a distributed system are built into Service Fabric, including:</span></span>

- <span data-ttu-id="56c4a-149">**Clusterverwaltung**:</span><span class="sxs-lookup"><span data-stu-id="56c4a-149">**Cluster management**.</span></span> <span data-ttu-id="56c4a-150">Service Fabric übernimmt automatisch Knotenfailover, Integritätsüberwachung und andere Funktionen der Clusterverwaltung.</span><span class="sxs-lookup"><span data-stu-id="56c4a-150">Service Fabric automatically handles node failover, health monitoring, and other cluster management functions.</span></span>
- <span data-ttu-id="56c4a-151">**Horizontale Skalierung**:</span><span class="sxs-lookup"><span data-stu-id="56c4a-151">**Horizontal scaling**.</span></span> <span data-ttu-id="56c4a-152">Wenn Sie Knoten zu einem Service Fabric-Cluster hinzufügen, wird die Anwendung automatisch skaliert, da die Dienste auf die neuen Knoten verteilt werden.</span><span class="sxs-lookup"><span data-stu-id="56c4a-152">When you add nodes to a Service Fabric cluster, the application automatically scales, as services are distributed across the new nodes.</span></span>
- <span data-ttu-id="56c4a-153">**Dienstermittlung**:</span><span class="sxs-lookup"><span data-stu-id="56c4a-153">**Service discovery**.</span></span> <span data-ttu-id="56c4a-154">Service Fabric stellt einen Ermittlungsdienst bereit, der den Endpunkt für einen benannten Dienst auflösen kann.</span><span class="sxs-lookup"><span data-stu-id="56c4a-154">Service Fabric provides a discovery service that can resolve the endpoint for a named service.</span></span>
- <span data-ttu-id="56c4a-155">**Zustandslose und zustandsbehaftete Dienste**:</span><span class="sxs-lookup"><span data-stu-id="56c4a-155">**Stateless and stateful services**.</span></span> <span data-ttu-id="56c4a-156">Zustandsbehaftete Dienste verwenden [zuverlässige Sammlungen][sf-reliable-collections], die an die Stelle eines Cache oder einer Warteschlange treten und partitioniert werden können.</span><span class="sxs-lookup"><span data-stu-id="56c4a-156">Stateful services use [reliable collections][sf-reliable-collections], which can take the place of a cache or queue, and can be partitioned.</span></span>
- <span data-ttu-id="56c4a-157">**Anwendungslebenszyklusverwaltung**:</span><span class="sxs-lookup"><span data-stu-id="56c4a-157">**Application lifecycle management**.</span></span> <span data-ttu-id="56c4a-158">Ein Upgrade der Dienste ist unabhängig und ohne Ausfallzeiten der Anwendungen möglich.</span><span class="sxs-lookup"><span data-stu-id="56c4a-158">Services can be upgraded independently and without application downtime.</span></span>
- <span data-ttu-id="56c4a-159">**Dienstorchestrierung** für einen Cluster aus Computern.</span><span class="sxs-lookup"><span data-stu-id="56c4a-159">**Service orchestration** across a cluster of machines.</span></span>
- <span data-ttu-id="56c4a-160">**Höhere Dichte** zur Optimierung des Ressourcenverbrauchs.</span><span class="sxs-lookup"><span data-stu-id="56c4a-160">**Higher density** for optimizing resource consumption.</span></span> <span data-ttu-id="56c4a-161">Ein einzelner Knoten kann mehrere Dienste hosten.</span><span class="sxs-lookup"><span data-stu-id="56c4a-161">A single node can host multiple services.</span></span>

<span data-ttu-id="56c4a-162">Service Fabric wird von verschiedenen Microsoft-Diensten verwendet, darunter Azure SQL-Datenbank, Cosmos DB, Azure Event Hubs und andere, was es zu einer bewährten Plattform für die Erstellung verteilter Cloudanwendungen macht.</span><span class="sxs-lookup"><span data-stu-id="56c4a-162">Service Fabric is used by various Microsoft services, including Azure SQL Database, Cosmos DB, Azure Event Hubs, and others, making it a proven platform for building distributed cloud applications.</span></span>
