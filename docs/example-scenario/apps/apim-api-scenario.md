2. <span data-ttu-id="74338-125">Aufrufe von der vorhandenen Web-App an die vorhandenen HTTP-Dienste bleiben unverändert.</span><span class="sxs-lookup"><span data-stu-id="74338-125">Calls from the existing web app to the existing HTTP services will remain unchanged.</span></span> <span data-ttu-id="74338-126">Diese Aufrufe erfolgen intern im Unternehmensnetzwerk.</span><span class="sxs-lookup"><span data-stu-id="74338-126">These calls are internal to the corporate network.</span></span>
3. <span data-ttu-id="74338-127">Eingehende Aufrufe werden von Azure an die vorhandenen internen Dienste gesendet:</span><span class="sxs-lookup"><span data-stu-id="74338-127">Inbound calls are made from Azure to the existing internal services:</span></span>
    * <span data-ttu-id="74338-128">Das Sicherheitsteam verwendet den [sicheren Transport (HTTPS/SSL)][apim-ssl], um zuzulassen, dass Datenverkehr von der APIM-Instanz durch die Unternehmensfirewall zu den vorhandenen lokalen Diensten geleitet wird.</span><span class="sxs-lookup"><span data-stu-id="74338-128">The security team allows traffic from the APIM instance to pass through the corporate firewall to the existing on-premises services [using secure transport (HTTPS/SSL)][apim-ssl].</span></span>
    * <span data-ttu-id="74338-129">Das operative Team lässt eingehende Aufrufe der Dienste nur von der APIM-Instanz zu.</span><span class="sxs-lookup"><span data-stu-id="74338-129">The operations team will allow inbound calls to the services only from the APIM instance.</span></span> <span data-ttu-id="74338-130">Innerhalb der Unternehmensnetzwerkumgebung wird die [IP-Adresse der APIM-Instanz auf die Whitelist gesetzt][apim-whitelist-ip], um diese Anforderung zu erfüllen.</span><span class="sxs-lookup"><span data-stu-id="74338-130">This requirement is met by [white-listing the IP address of the APIM instance][apim-whitelist-ip] within the corporate network perimeter.</span></span>

1. <span data-ttu-id="74338-132">Für die neue API gilt Folgendes:</span><span class="sxs-lookup"><span data-stu-id="74338-132">The new API:</span></span>
    * <span data-ttu-id="74338-133">Die neue API wird nur über die APIM-Instanz verfügbar gemacht, die die API-Fassade bereitstellt.</span><span class="sxs-lookup"><span data-stu-id="74338-133">Is surfaced only through the APIM instance, which will provide the API facade.</span></span> <span data-ttu-id="74338-134">Auf die neue API wird nicht direkt zugegriffen.</span><span class="sxs-lookup"><span data-stu-id="74338-134">The new API won't be accessed directly.</span></span>
    * <span data-ttu-id="74338-135">Die neue API wird als eine [Azure-PaaS-Web-API-App][azure-api-apps] entwickelt und veröffentlicht.</span><span class="sxs-lookup"><span data-stu-id="74338-135">Is developed and published as an [Azure PaaS Web API App][azure-api-apps].</span></span>
    * <span data-ttu-id="74338-136">Die neue API wird auf die Whitelist gesetzt (über [Web-App-Einstellungen][azure-appservice-ip-restrict]), um nur die [virtuelle IP (VIP) der APIM-Instanz][apim-faq-vip] zu akzeptieren.</span><span class="sxs-lookup"><span data-stu-id="74338-136">Is white-listed (via [Web App settings][azure-appservice-ip-restrict]) to accept only the [APIM VIP][apim-faq-vip].</span></span>
    * <span data-ttu-id="74338-137">Die neue API wird mit aktiviertem sicheren Transport/SSL-Protokoll in Azure-Web-Apps gehostet.</span><span class="sxs-lookup"><span data-stu-id="74338-137">Is hosted in Azure Web Apps with Secure Transport/SSL turned on.</span></span>
    * <span data-ttu-id="74338-138">Für die neue API ist die Autorisierung aktiviert. Diese wird von [Azure App Service][azure-appservice-auth] mithilfe von Azure Active Directory und OAuth2 bereitgestellt.</span><span class="sxs-lookup"><span data-stu-id="74338-138">Has authorization enabled, [provided by the Azure App Service][azure-appservice-auth] using Azure Active Directory and OAuth2.</span></span>
2. <span data-ttu-id="74338-139">Die neue browserbasierte Webanwendung ist für die vorhandene HTTP-API **und** die neue API von der Azure API Management-Instanz abhängig.</span><span class="sxs-lookup"><span data-stu-id="74338-139">The new browser-based web application will depend on the Azure API Management instance for **both** the existing HTTP API and the new API.</span></span>

<span data-ttu-id="74338-140">Die APIM-Instanz wird so konfiguriert, dass sie die älteren HTTP-Dienste einem neuen API-Vertrag zuordnet.</span><span class="sxs-lookup"><span data-stu-id="74338-140">The APIM instance will be configured to map the legacy HTTP services to a new API contract.</span></span> <span data-ttu-id="74338-141">Der neuen Webbenutzeroberfläche ist die Integration einer Gruppe von älteren Diensten/APIs und neuen APIs daher nicht bekannt.</span><span class="sxs-lookup"><span data-stu-id="74338-141">By doing this, the new Web UI is unaware of the integration with a set of legacy services/APIs and new APIs.</span></span> <span data-ttu-id="74338-142">Das Projektteam wird in der Zukunft schrittweise Funktionen zu den neuen APIs portieren und die ursprünglichen Dienste außer Betrieb setzen.</span><span class="sxs-lookup"><span data-stu-id="74338-142">In the future, the project team will gradually port functionality to the new APIs and retire the original services.</span></span> <span data-ttu-id="74338-143">Diese Änderungen werden in der APIM-Konfiguration vorgenommen, sodass die Front-End-Benutzeroberfläche davon nicht betroffen ist und Neuentwicklungen vermieden werden.</span><span class="sxs-lookup"><span data-stu-id="74338-143">These changes will be handled within APIM configuration, leaving the front-end UI unaffected and avoiding redevelopment work.</span></span>

  * <span data-ttu-id="74338-148">Im lokalen Szenario könnte die API Management-Instanz eine private Verbindung über [ein Azure-VPN-Gateway und eine Site-to-Site-IPSec-VPN-Verbindung][azure-vpn] oder [ ExpressRoute][azure-er] mit dem internen Dienst herstellen, wodurch ein [Hybridszenario mit Azure und einer lokalen Umgebung][azure-hybrid] entstehen würde.</span><span class="sxs-lookup"><span data-stu-id="74338-148">In the on-premises scenario, the API Management instance could reach back to the internal service privately via an [Azure VPN gateway and site-to-site IPSec VPN connection][azure-vpn] or [ExpressRoute][azure-er] making this a [hybrid Azure and on-premises scenario][azure-hybrid].</span></span>
