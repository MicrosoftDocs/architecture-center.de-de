<span data-ttu-id="50bed-109">Schichten sind physisch getrennt und befinden sich auf verschiedenen Computern.</span><span class="sxs-lookup"><span data-stu-id="50bed-109">Tiers are physically separated, running on separate machines.</span></span> <span data-ttu-id="50bed-110">Eine Schicht kann eine andere Schicht direkt aufrufen oder asynchrones Messaging verwenden (Nachrichtenwarteschlange).</span><span class="sxs-lookup"><span data-stu-id="50bed-110">A tier can call to another tier directly, or use asynchronous messaging (message queue).</span></span> <span data-ttu-id="50bed-111">Jede Ebene kann auf einer eigenen Schicht gehostet werden, dies ist jedoch nicht erforderlich.</span><span class="sxs-lookup"><span data-stu-id="50bed-111">Although each layer might be hosted in its own tier, that's not required.</span></span> <span data-ttu-id="50bed-112">Verschiedene Ebenen können auf der gleichen Schicht gehostet werden.</span><span class="sxs-lookup"><span data-stu-id="50bed-112">Several layers might be hosted on the same tier.</span></span> <span data-ttu-id="50bed-113">Die physische Trennung der Schichten verbessert die Skalierbarkeit und Resilienz, erhöht jedoch aufgrund der zusätzlichen Netzwerkkommunikation die Latenz.</span><span class="sxs-lookup"><span data-stu-id="50bed-113">Physically separating the tiers improves scalability and resiliency, but also adds latency from the additional network communication.</span></span>

<span data-ttu-id="50bed-118">Eine n-schichtige Anwendung kann auf einer **Architektur mit geschlossenen Ebenen** oder einer **Architektur mit offenen Ebenen** basieren:</span><span class="sxs-lookup"><span data-stu-id="50bed-118">An N-tier application can have a **closed layer architecture** or an **open layer architecture**:</span></span>
