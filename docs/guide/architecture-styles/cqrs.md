# <a name="cqrs-architecture-style"></a><span data-ttu-id="c4080-103">Architekturstil „CQRS“</span><span class="sxs-lookup"><span data-stu-id="c4080-103">CQRS architecture style</span></span>

<span data-ttu-id="c4080-104">CQRS (Command and Query Responsibility Segregation, Befehlsabfrage-Zuständigkeitstrennung) ist ein Architekturstil, bei dem Lesevorgänge von Schreibvorgängen getrennt werden.</span><span class="sxs-lookup"><span data-stu-id="c4080-104">Command and Query Responsibility Segregation (CQRS) is an architecture style that separates read operations from write operations.</span></span>

![Logisches Diagramm einer CQRS-Architektur](./images/cqrs-logical.svg)

<span data-ttu-id="c4080-106">In herkömmlichen Architekturen wird zum Abfragen und Aktualisieren einer Datenbank das gleiche Datenmodell verwendet.</span><span class="sxs-lookup"><span data-stu-id="c4080-106">In traditional architectures, the same data model is used to query and update a database.</span></span> <span data-ttu-id="c4080-107">Dies ist eine einfache Vorgehensweise, die für grundlegende CRUD-Vorgänge gut geeignet ist.</span><span class="sxs-lookup"><span data-stu-id="c4080-107">That's simple and works well for basic CRUD operations.</span></span> <span data-ttu-id="c4080-108">Bei komplexeren Anwendungen kann dieser Ansatz aber zu Problemen führen.</span><span class="sxs-lookup"><span data-stu-id="c4080-108">In more complex applications, however, this approach can become unwieldy.</span></span> <span data-ttu-id="c4080-109">Beispielsweise werden von der Anwendung auf Leseseite unter Umständen unterschiedliche Abfragen durchgeführt und Datenübertragungsobjekte (Data Transfer Objects, DTOs) mit uneinheitlichen Formen zurückgegeben.</span><span class="sxs-lookup"><span data-stu-id="c4080-109">For example, on the read side, the application may perform many different queries, returning data transfer objects (DTOs) with different shapes.</span></span> <span data-ttu-id="c4080-110">Die Objektzuordnung kann kompliziert werden.</span><span class="sxs-lookup"><span data-stu-id="c4080-110">Object mapping can become complicated.</span></span> <span data-ttu-id="c4080-111">Auf der Schreibseite werden für das Modell ggf. eine komplexe Überprüfung und Geschäftslogik implementiert.</span><span class="sxs-lookup"><span data-stu-id="c4080-111">On the write side, the model may implement complex validation and business logic.</span></span> <span data-ttu-id="c4080-112">Hieraus kann sich ein Modell mit zu hoher Komplexität ergeben, das überdimensioniert ist.</span><span class="sxs-lookup"><span data-stu-id="c4080-112">As a result, you can end up with an overly complex model that does too much.</span></span>

<span data-ttu-id="c4080-113">Ein anderes potenzielles Problem ist, dass Lese- und Schreibworkloads häufig asymmetrisch sind und über sehr unterschiedliche Leistungs- und Skalierungsanforderungen verfügen.</span><span class="sxs-lookup"><span data-stu-id="c4080-113">Another potential problem is that read and write workloads are often asymmetrical, with very different performance and scale requirements.</span></span>

<span data-ttu-id="c4080-114">CQRS löst diese Probleme, indem Lese- und Schreibvorgänge in separate Modelle unterteilt werden. Hierfür werden **Befehle** zum Aktualisieren von Daten und **Abfragen** zum Lesen von Daten verwendet.</span><span class="sxs-lookup"><span data-stu-id="c4080-114">CQRS addresses these problems by separating reads and writes into separate models, using **commands** to update data, and **queries** to read data.</span></span>

- <span data-ttu-id="c4080-115">Die Befehle sollten nicht datenzentriert, sondern aufgabenbasiert sein.</span><span class="sxs-lookup"><span data-stu-id="c4080-115">Commands should be task based, rather than data centric.</span></span> <span data-ttu-id="c4080-116">(Beispiel: „Hotelzimmer buchen“, nicht „Reservierungsstatus auf ‚Reserviert‘ festlegen“.) Befehle können in eine Warteschlange für die asynchrone Verarbeitung eingereiht werden, anstatt synchron verarbeitet zu werden.</span><span class="sxs-lookup"><span data-stu-id="c4080-116">("Book hotel room," not "set ReservationStatus to Reserved.") Commands may be placed on a queue for asynchronous processing, rather than being processed synchronously.</span></span>

- <span data-ttu-id="c4080-117">Mit Abfragen wird die Datenbank niemals geändert.</span><span class="sxs-lookup"><span data-stu-id="c4080-117">Queries never modify the database.</span></span> <span data-ttu-id="c4080-118">Eine Abfrage gibt ein DTO zurück, in dem kein Domänenwissen gekapselt ist.</span><span class="sxs-lookup"><span data-stu-id="c4080-118">A query returns a DTO that does not encapsulate any domain knowledge.</span></span>

<span data-ttu-id="c4080-119">Eine stärkere Isolation erzielen Sie, indem Sie die Lesedaten physisch von den Schreibdaten trennen.</span><span class="sxs-lookup"><span data-stu-id="c4080-119">For greater isolation, you can physically separate the read data from the write data.</span></span> <span data-ttu-id="c4080-120">In diesem Fall kann die Lesedatenbank ihr eigenes Datenschema verwenden, das für Abfragen optimiert ist.</span><span class="sxs-lookup"><span data-stu-id="c4080-120">In that case, the read database can use its own data schema that is optimized for queries.</span></span> <span data-ttu-id="c4080-121">Beispielsweise kann eine [materialisierte Sicht][materialized-view] der Daten gespeichert werden, um komplexe Verknüpfungen oder O/RM-Zuordnungen zu vermeiden.</span><span class="sxs-lookup"><span data-stu-id="c4080-121">For example, it can store a [materialized view][materialized-view] of the data, in order to avoid complex joins or complex O/RM mappings.</span></span> <span data-ttu-id="c4080-122">Es kann sogar ein anderer Typ von Datenspeicher verwendet werden.</span><span class="sxs-lookup"><span data-stu-id="c4080-122">It might even use a different type of data store.</span></span> <span data-ttu-id="c4080-123">Die Schreibdatenbank kann beispielsweise relational sein, während es sich bei der Lesedatenbank um eine Dokumentdatenbank handelt.</span><span class="sxs-lookup"><span data-stu-id="c4080-123">For example, the write database might be relational, while the read database is a document database.</span></span>

<span data-ttu-id="c4080-124">Wenn separate Lese- und Schreibdatenbanken verwendet werden, müssen sie synchron gehalten werden. Dies wird normalerweise erreicht, indem das Schreibmodell jeweils ein Ereignis veröffentlicht, wenn es die Datenbank aktualisiert.</span><span class="sxs-lookup"><span data-stu-id="c4080-124">If separate read and write databases are used, they must be kept in sync. Typically this is accomplished by having the write model publish an event whenever it updates the database.</span></span> <span data-ttu-id="c4080-125">Die Aktualisierung der Datenbank und die Veröffentlichung des Ereignisses müssen in einer einzelnen Transaktion durchgeführt werden.</span><span class="sxs-lookup"><span data-stu-id="c4080-125">Updating the database and publishing the event must occur in a single transaction.</span></span>

<span data-ttu-id="c4080-126">Für einige Implementierungen von CQRS wird das [Muster für die Ereignisherkunftsermittlung][event-sourcing] verwendet.</span><span class="sxs-lookup"><span data-stu-id="c4080-126">Some implementations of CQRS use the [Event Sourcing pattern][event-sourcing].</span></span> <span data-ttu-id="c4080-127">Bei diesem Muster wird der Anwendungsstatus als Ereignissequenz gespeichert.</span><span class="sxs-lookup"><span data-stu-id="c4080-127">With this pattern, application state is stored as a sequence of events.</span></span> <span data-ttu-id="c4080-128">Jedes Ereignis umfasst eine Reihe von Änderungen der Daten.</span><span class="sxs-lookup"><span data-stu-id="c4080-128">Each event represents a set of changes to the data.</span></span> <span data-ttu-id="c4080-129">Der aktuelle Zustand wird hergestellt, indem die Ereignisse erneut durchgeführt werden.</span><span class="sxs-lookup"><span data-stu-id="c4080-129">The current state is constructed by replaying the events.</span></span> <span data-ttu-id="c4080-130">In einem CQRS-Kontext besteht ein Vorteil der Ereignisherkunftsermittlung darin, dass dieselben Ereignisse verwendet werden können, um andere Komponenten zu benachrichtigen, z.B. vor allem das Lesemodell.</span><span class="sxs-lookup"><span data-stu-id="c4080-130">In a CQRS context, one benefit of Event Sourcing is that the same events can be used to notify other components &mdash; in particular, to notify the read model.</span></span> <span data-ttu-id="c4080-131">Im Lesemodell werden die Ereignisse genutzt, um eine Momentaufnahme des aktuellen Zustands zu erstellen, da dies für Abfragen effizienter ist.</span><span class="sxs-lookup"><span data-stu-id="c4080-131">The read model uses the events to create a snapshot of the current state, which is more efficient for queries.</span></span> <span data-ttu-id="c4080-132">Mit der Ereignisherkunftsermittlung erhöht sich aber auch die Komplexität des Designs.</span><span class="sxs-lookup"><span data-stu-id="c4080-132">However, Event Sourcing adds complexity to the design.</span></span>

## <a name="when-to-use-this-architecture"></a><span data-ttu-id="c4080-134">Einsatzmöglichkeiten für diese Architektur</span><span class="sxs-lookup"><span data-stu-id="c4080-134">When to use this architecture</span></span>

<span data-ttu-id="c4080-135">Erwägen Sie den Einsatz von CQRS für Kollaborationsdomänen, in denen viele Benutzer auf die gleichen Daten zugreifen – vor allem, wenn die Lese- und Schreibworkloads asymmetrisch sind.</span><span class="sxs-lookup"><span data-stu-id="c4080-135">Consider CQRS for collaborative domains where many users access the same data, especially when the read and write workloads are asymmetrical.</span></span>

<span data-ttu-id="c4080-136">CQRS ist keine Architektur der obersten Ebene, die für ein gesamtes System gilt.</span><span class="sxs-lookup"><span data-stu-id="c4080-136">CQRS is not a top-level architecture that applies to an entire system.</span></span> <span data-ttu-id="c4080-137">Wenden Sie CQRS nur auf die Subsysteme an, wenn das Trennen von Lese- und Schreibvorgängen einen eindeutigen Nutzen hat.</span><span class="sxs-lookup"><span data-stu-id="c4080-137">Apply CQRS only to those subsystems where there is clear value in separating reads and writes.</span></span> <span data-ttu-id="c4080-138">Andernfalls erhöhen Sie die Komplexität, ohne dass sich daraus ein Vorteil ergibt.</span><span class="sxs-lookup"><span data-stu-id="c4080-138">Otherwise, you are creating additional complexity for no benefit.</span></span>

## <a name="benefits"></a><span data-ttu-id="c4080-139">Vorteile</span><span class="sxs-lookup"><span data-stu-id="c4080-139">Benefits</span></span>

- <span data-ttu-id="c4080-140">**Unabhängige Skalierung**:</span><span class="sxs-lookup"><span data-stu-id="c4080-140">**Independently scaling**.</span></span> <span data-ttu-id="c4080-141">CQRS ermöglicht das unabhängige Skalieren der Lese- und Schreibworkloads und kann zu einer Verringerung von Sperrkonflikten führen.</span><span class="sxs-lookup"><span data-stu-id="c4080-141">CQRS allows the read and write workloads to scale independently, and may result in fewer lock contentions.</span></span>
- <span data-ttu-id="c4080-142">**Optimierte Datenschemas**:</span><span class="sxs-lookup"><span data-stu-id="c4080-142">**Optimized data schemas**.</span></span> <span data-ttu-id="c4080-143">Auf der Leseseite kann ein Schema verwendet werden, das für Abfragen optimiert ist, während auf der Schreibseite ein Schema verwendet wird, das für Updates optimiert ist.</span><span class="sxs-lookup"><span data-stu-id="c4080-143">The read side can use a schema that is optimized for queries, while the write side uses a schema that is optimized for updates.</span></span>
- <span data-ttu-id="c4080-144">**Sicherheit**.</span><span class="sxs-lookup"><span data-stu-id="c4080-144">**Security**.</span></span> <span data-ttu-id="c4080-145">Es ist einfacher sicherzustellen, dass nur die richtigen Domänenentitäten Schreibvorgänge für die Daten durchführen.</span><span class="sxs-lookup"><span data-stu-id="c4080-145">It's easier to ensure that only the right domain entities are performing writes on the data.</span></span>
- <span data-ttu-id="c4080-146">**Trennung von Zuständigkeiten**:</span><span class="sxs-lookup"><span data-stu-id="c4080-146">**Separation of concerns**.</span></span> <span data-ttu-id="c4080-147">Das Trennen der Lese- und Schreibseite kann zu Modellen führen, die besser gewartet werden können und flexibler sind.</span><span class="sxs-lookup"><span data-stu-id="c4080-147">Segregating the read and write sides can result in models that are more maintainable and flexible.</span></span> <span data-ttu-id="c4080-148">Der größte Teil der komplexen Geschäftslogik betrifft das Schreibmodell.</span><span class="sxs-lookup"><span data-stu-id="c4080-148">Most of the complex business logic goes into the write model.</span></span> <span data-ttu-id="c4080-149">Das Lesemodell kann relativ einfach gestaltet sein.</span><span class="sxs-lookup"><span data-stu-id="c4080-149">The read model can be relatively simple.</span></span>
- <span data-ttu-id="c4080-150">**Einfachere Abfragen**:</span><span class="sxs-lookup"><span data-stu-id="c4080-150">**Simpler queries**.</span></span> <span data-ttu-id="c4080-151">Indem eine materialisierte Sicht in der Lesedatenbank gespeichert wird, kann die Anwendung beim Durchführen von Abfragen komplexe Verknüpfungen vermeiden.</span><span class="sxs-lookup"><span data-stu-id="c4080-151">By storing a materialized view in the read database, the application can avoid complex joins when querying.</span></span>

- <span data-ttu-id="c4080-153">**Komplexität**.</span><span class="sxs-lookup"><span data-stu-id="c4080-153">**Complexity**.</span></span> <span data-ttu-id="c4080-154">Die Grundidee von CQRS ist einfach.</span><span class="sxs-lookup"><span data-stu-id="c4080-154">The basic idea of CQRS is simple.</span></span> <span data-ttu-id="c4080-155">Es kann sich aber auch ein komplexeres Anwendungsdesign ergeben, z.B. bei Verwendung des Musters für die Ereignisherkunftsermittlung.</span><span class="sxs-lookup"><span data-stu-id="c4080-155">But it can lead to a more complex application design, especially if they include the Event Sourcing pattern.</span></span>
