## <a name="tagging"></a><span data-ttu-id="8a5ce-113">Kennzeichnung (Tagging)</span><span class="sxs-lookup"><span data-stu-id="8a5ce-113">Tagging</span></span>

<span data-ttu-id="8a5ce-114">**Technisches Risiko**: Ohne die Zuordnung geeigneter Metadatentags zu den bereitgestellten Ressourcen kann der IT-Betrieb die Unterstützung oder Optimierung von Ressourcen auf Grundlage der erforderlichen Vereinbarung zum Servicelevel, der Wichtigkeit von Geschäftsvorgängen oder der Betriebskosten nicht priorisieren.</span><span class="sxs-lookup"><span data-stu-id="8a5ce-114">**Technical risk**: Without proper metadata tagging associated with deployed resources, IT Operations cannot prioritize support or optimization of resources based on required SLA, importance to business operations, or operational cost.</span></span> <span data-ttu-id="8a5ce-115">Dies kann zur fehlerhaften Zuordnung von IT-Ressourcen und potenziellen Verzögerungen bei der Behebung von Vorfällen führen.</span><span class="sxs-lookup"><span data-stu-id="8a5ce-115">This can result in mis-allocation of IT resources and potential delays in incident resolution.</span></span>

<span data-ttu-id="8a5ce-116">**Richtlinienanweisung**: Die folgenden Richtlinien werden implementiert:</span><span class="sxs-lookup"><span data-stu-id="8a5ce-116">**Policy statement**: The following policies will be implemented:</span></span>

- <span data-ttu-id="8a5ce-117">Bereitgestellte Ressourcen sollten mit den folgenden Werten gekennzeichnet werden: Kosten, Kritikalität, SLA und Umgebung.</span><span class="sxs-lookup"><span data-stu-id="8a5ce-117">Deployed assets should be tagged with the following values: cost, criticality, SLA, and environment.</span></span>
- <span data-ttu-id="8a5ce-118">Mit Governancetools muss die Kennzeichnung (Tagging) im Hinblick auf Kosten, Kritikalität, SLA, Anwendung und Umgebung überprüft werden.</span><span class="sxs-lookup"><span data-stu-id="8a5ce-118">Governance tooling must validate tagging related to cost, criticality, SLA, application, and environment.</span></span> <span data-ttu-id="8a5ce-119">Alle Werte müssen sich an vordefinierten Werten ausrichten, die vom Governance-Team verwaltet werden.</span><span class="sxs-lookup"><span data-stu-id="8a5ce-119">All values must align to predefined values managed by the governance team.</span></span>

<span data-ttu-id="8a5ce-120">**Potenzielle Entwurfsoptionen**: In Azure werden für die meisten Ressourcentypen [Standardmetadatentags der Form „Name-Wert“](/azure/azure-resource-manager/resource-group-using-tags) unterstützt.</span><span class="sxs-lookup"><span data-stu-id="8a5ce-120">**Potential design options**: In Azure, [standard name-value metadata tags](/azure/azure-resource-manager/resource-group-using-tags) are supported on most resource types.</span></span> <span data-ttu-id="8a5ce-121">[Azure Policy](/azure/governance/policy/overview) wird verwendet, um bestimmte Tags im Rahmen der Erstellung von Ressourcen zu erzwingen.</span><span class="sxs-lookup"><span data-stu-id="8a5ce-121">[Azure Policy](/azure/governance/policy/overview) is used to enforce specific tags as part of resource creation.</span></span>

## <a name="ungoverned-subscriptions"></a><span data-ttu-id="8a5ce-122">Ungeregelte Abonnements</span><span class="sxs-lookup"><span data-stu-id="8a5ce-122">Ungoverned subscriptions</span></span>

<span data-ttu-id="8a5ce-123">**Technisches Risiko**: Das beliebige Erstellen von Abonnements und Verwaltungsgruppen kann zu isolierten Abschnitten Ihrer Cloudumgebung führen, die nicht ordnungsgemäß Ihren Governance-Richtlinien unterliegen.</span><span class="sxs-lookup"><span data-stu-id="8a5ce-123">**Technical risk**: Arbitrary creation of subscriptions and management groups can lead to isolated sections of your cloud estate that are not properly subject to your governance policies.</span></span>

<span data-ttu-id="8a5ce-124">**Richtlinienanweisung**: Für die Erstellung neuer Abonnements oder Verwaltungsgruppen für unternehmenskritische Anwendungen oder geschützte Daten ist eine Überprüfung durch das Cloud Governance-Team erforderlich.</span><span class="sxs-lookup"><span data-stu-id="8a5ce-124">**Policy statement**: Creation of new subscriptions or management groups for any mission-critical applications or protected data will require a review from the Cloud Governance team.</span></span> <span data-ttu-id="8a5ce-125">Genehmigte Änderungen werden in eine ordnungsgemäße Blaupausenzuweisung integriert.</span><span class="sxs-lookup"><span data-stu-id="8a5ce-125">Approved changes will be integrated into a proper blueprint assignment.</span></span>

<span data-ttu-id="8a5ce-126">**Potenzielle Entwurfsoptionen**: Schränken Sie den administrativen Zugriffs auf die [Azure-Verwaltungsgruppen](/azure/governance/management-groups/) Ihrer Organisation auf nur genehmigte Governance-Teammitglieder ein, die das Erstellen von Abonnements und den Zugriffssteuerungsprozess kontrollieren.</span><span class="sxs-lookup"><span data-stu-id="8a5ce-126">**Potential design options**: Lock down administrative access to your organizations [Azure management groups](/azure/governance/management-groups/) to only approved governance team members who will control the subscription creation and access control process.</span></span>

## <a name="manage-updates-to-virtual-machines"></a><span data-ttu-id="8a5ce-127">Verwalten von Aktualisierungen für virtuelle Computer</span><span class="sxs-lookup"><span data-stu-id="8a5ce-127">Manage updates to virtual machines</span></span>

<span data-ttu-id="8a5ce-128">**Technisches Risiko**: Virtuelle Computer (VMs), die nicht auf dem neuesten Stand hinsichtlich neuester Updates und Softwarepatches sind, sind anfällig für Sicherheits- oder Leistungsprobleme, die zu Dienstunterbrechungen führen können.</span><span class="sxs-lookup"><span data-stu-id="8a5ce-128">**Technical risk**: Virtual machines (VMs) that are not up-to-date with the latest updates and software patches are vulnerable to security or performance issues, which can result in service disruptions.</span></span>

<span data-ttu-id="8a5ce-129">**Richtlinienanweisung**: Governancetools müssen erzwingen, dass automatische Updates für alle bereitgestellten VMs aktiviert sind.</span><span class="sxs-lookup"><span data-stu-id="8a5ce-129">**Policy statement**: Governance tooling must enforce that automatic updates are enabled on all deployed VMs.</span></span> <span data-ttu-id="8a5ce-130">Verstöße müssen von Betriebsmanagementteams überprüft und in Übereinstimmung mit den Betriebsrichtlinien beseitigt werden.</span><span class="sxs-lookup"><span data-stu-id="8a5ce-130">Violations must be reviewed with operational management teams and remediated in accordance with operations policies.</span></span> <span data-ttu-id="8a5ce-131">Ressourcen, die nicht automatisch aktualisiert werden, müssen in Prozesse des IT-Betriebs einbezogen werden.</span><span class="sxs-lookup"><span data-stu-id="8a5ce-131">Assets that are not automatically updated must be included in processes owned by IT Operations.</span></span>

<span data-ttu-id="8a5ce-132">**Potenzielle Entwurfsoptionen**: Für in Azure gehostete VMs können konsistente Updateverwaltung bereitstellen, indem Sie die [Updateverwaltungslösung in Azure Automation](/azure/automation/automation-update-management) verwenden.</span><span class="sxs-lookup"><span data-stu-id="8a5ce-132">**Potential design options**: For Azure hosted VMs, you can provide consistent update management using the [Update Management solution in Azure Automation](/azure/automation/automation-update-management).</span></span>

## <a name="deployment-compliance"></a><span data-ttu-id="8a5ce-133">Bereitstellungscompliance</span><span class="sxs-lookup"><span data-stu-id="8a5ce-133">Deployment compliance</span></span>

<span data-ttu-id="8a5ce-134">**Technisches Risiko**: Bereitstellungsskripts und Automatisierungstools, die vom Cloud Governance-Team nicht vollständig geprüft wurden, können zu Ressourcenbereitstellungen führen, die gegen Richtlinien verstoßen.</span><span class="sxs-lookup"><span data-stu-id="8a5ce-134">**Technical risk**: Deployment scripts and automation tooling that is not fully vetted by the Cloud Governance team can result in resource deployments that violate policy.</span></span>

<span data-ttu-id="8a5ce-135">**Richtlinienanweisung**: Die folgenden Richtlinien werden implementiert:</span><span class="sxs-lookup"><span data-stu-id="8a5ce-135">**Policy statement**: The following policies will be implemented:</span></span>

<span data-ttu-id="8a5ce-140">**Technisches Risiko**: Nicht ordnungsgemäß implementierte oder inkonsistent instrumentierte Überwachung kann die Erkennung von Integritätsproblemen bei Workloads oder anderen Verstößen gegen die Richtliniencompliance verhindern.</span><span class="sxs-lookup"><span data-stu-id="8a5ce-140">**Technical risk**: Improperly implemented or inconsistently instrumented monitoring can prevent the detection of workload health issues or other policy compliance violations.</span></span>
