## <a name="basic-grouping"></a><span data-ttu-id="e03e0-116">Einfache Gruppierung</span><span class="sxs-lookup"><span data-stu-id="e03e0-116">Basic grouping</span></span>

<span data-ttu-id="e03e0-117">In Azure sind [Ressourcengruppen](/azure/azure-resource-manager/resource-group-overview#resource-groups) ein zentraler Mechanismus der Ressourcenorganisation, um Ressourcen innerhalb eines Abonnements logisch zu gruppieren.</span><span class="sxs-lookup"><span data-stu-id="e03e0-117">In Azure, [resource groups](/azure/azure-resource-manager/resource-group-overview#resource-groups) are a core resource organization mechanism to logically group resources within a subscription.</span></span>

<span data-ttu-id="e03e0-118">Ressourcengruppen fungieren als Container für Ressourcen mit einem gemeinsamen Lebenszyklus oder gemeinsamen Verwaltungseinschränkungen wie Richtlinien oder Anforderungen an die rollenbasierte Zugriffssteuerung (RBAC).</span><span class="sxs-lookup"><span data-stu-id="e03e0-118">Resource groups act as containers for resources with a common lifecycle or shared management constraints such as policy or role-based access control (RBAC) requirements.</span></span> <span data-ttu-id="e03e0-119">Ressourcengruppen dürfen nicht geschachtelt sein, und Ressourcen dürfen nur zu einer einzigen Ressourcengruppe gehören.</span><span class="sxs-lookup"><span data-stu-id="e03e0-119">Resource groups can't be nested, and resources can only belong to one resource group.</span></span> <span data-ttu-id="e03e0-120">Einige Aktionen gelten für alle Ressourcen in einer Ressourcengruppe.</span><span class="sxs-lookup"><span data-stu-id="e03e0-120">Some actions can act on all resources in a resource group.</span></span> <span data-ttu-id="e03e0-121">Durch das Löschen einer Ressourcengruppe werden beispielsweise alle Ressourcen aus dieser Gruppe entfernt.</span><span class="sxs-lookup"><span data-stu-id="e03e0-121">For example, deleting a resource group removes all resources within that group.</span></span> <span data-ttu-id="e03e0-122">Zum Erstellen von Ressourcengruppen gibt es gängige Muster, die üblicherweise in zwei Kategorien aufgeteilt sind:</span><span class="sxs-lookup"><span data-stu-id="e03e0-122">There are common patterns when creating resource groups, commonly divided into two categories:</span></span>

- <span data-ttu-id="e03e0-123">Herkömmliche IT-Workloads: Werden meist nach Elementen innerhalb des gleichen Lebenszyklus gruppiert (z.B. Anwendung).</span><span class="sxs-lookup"><span data-stu-id="e03e0-123">Traditional IT workloads: Most often grouped by items within the same lifecycle, such as an application.</span></span> <span data-ttu-id="e03e0-124">Eine Gruppierung nach Anwendung ermöglicht die Verwaltung einzelner Anwendungen.</span><span class="sxs-lookup"><span data-stu-id="e03e0-124">Grouping by application allows for individual application management.</span></span>
- <span data-ttu-id="e03e0-125">Agile IT-Workloads: Konzentrieren sich auf Cloudanwendungen, die externen Kunden zugänglich sind.</span><span class="sxs-lookup"><span data-stu-id="e03e0-125">Agile IT workloads: Focus on external customer-facing cloud applications.</span></span> <span data-ttu-id="e03e0-126">Diese Ressourcengruppen spiegeln häufig die Funktionsebenen der Bereitstellung (z.B. Webschicht, App-Schicht) und der Verwaltung wider.</span><span class="sxs-lookup"><span data-stu-id="e03e0-126">These resource groups often reflect the functional layers of deployment (such as web tier or app tier) and management.</span></span>

## <a name="deployment-consistency"></a><span data-ttu-id="e03e0-127">Bereitstellungskonsistenz</span><span class="sxs-lookup"><span data-stu-id="e03e0-127">Deployment consistency</span></span>

<span data-ttu-id="e03e0-128">Aufbauend auf dem grundlegenden Mechanismus zur Gruppierung von Ressourcen bieten die meisten Cloudplattformen ein System zur Verwendung von Vorlagen für die Bereitstellung Ihrer Ressourcen in der Cloudumgebung.</span><span class="sxs-lookup"><span data-stu-id="e03e0-128">Building on top of the base resource grouping mechanism, most cloud platforms provide a system for using templates to deploy your resources to the cloud environment.</span></span> <span data-ttu-id="e03e0-129">Sie können Vorlagen verwenden, um konsistente Organisations- und Benennungskonventionen bei der Bereitstellung von Workloads zu erstellen und diese Aspekte der Ressourcenbereitstellung und des Verwaltungskonzepts zu erzwingen.</span><span class="sxs-lookup"><span data-stu-id="e03e0-129">You can use templates to create consistent organization and naming conventions when deploying workloads, enforcing those aspects of your resource deployment and management design.</span></span>

<span data-ttu-id="e03e0-130">[Azure Resource Manager-Vorlagen](/azure/azure-resource-manager/resource-group-overview#template-deployment) ermöglichen Ihnen, Ihre Ressourcen unter Verwendung einer vorgegebenen Konfigurations- und Ressourcengruppenstruktur wiederholt in konsistentem Zustand bereitzustellen.</span><span class="sxs-lookup"><span data-stu-id="e03e0-130">[Azure Resource Manager templates](/azure/azure-resource-manager/resource-group-overview#template-deployment) allow you to repeatedly deploy your resources in a consistent state using a predetermined configuration and resource group structure.</span></span> <span data-ttu-id="e03e0-131">Resource Manager-Vorlagen helfen Ihnen, eine Reihe von Standards als Grundlage für Ihre Bereitstellungen zu definieren.</span><span class="sxs-lookup"><span data-stu-id="e03e0-131">Resource Manager templates help you define a set of standards as a basis for your deployments.</span></span>

<span data-ttu-id="e03e0-132">Beispielsweise können Sie eine Standardvorlage für die Bereitstellung einer Webserver-Workload nutzen, die zwei virtuelle Computer als Webserver enthält, kombiniert mit einem Lastenausgleich zur Verarbeitung des Datenverkehrs zwischen den Servern.</span><span class="sxs-lookup"><span data-stu-id="e03e0-132">For example, you can have a standard template for deploying a web server workload that contains two virtual machines as web servers combined with a load balancer to manage traffic between the servers.</span></span> <span data-ttu-id="e03e0-133">Sie können diese Vorlage anschließend wiederverwenden, um strukturell identische Bereitstellungen immer dann zu erstellen, wenn eine neue Webserver-Workload erforderlich ist, wobei nur der Bereitstellungsname und die zugehörigen IP-Adressen geändert werden.</span><span class="sxs-lookup"><span data-stu-id="e03e0-133">You can then reuse this template to create structurally identical deployments whenever a new web server workload is needed, only changing the deployment name and IP addresses involved.</span></span>

<span data-ttu-id="e03e0-134">Beachten Sie, dass Sie diese Vorlagen auch programmgesteuert bereitstellen und in Ihre CI-/CD-Systeme integrieren können.</span><span class="sxs-lookup"><span data-stu-id="e03e0-134">Note that you can also programmatically deploy these templates and integrate them with your CI/CD systems.</span></span>

## <a name="policy-consistency"></a><span data-ttu-id="e03e0-135">Richtlinienkonsistenz</span><span class="sxs-lookup"><span data-stu-id="e03e0-135">Policy consistency</span></span>

<span data-ttu-id="e03e0-136">Um sicherzustellen, dass Governancerichtlinien bei der Erstellung von Ressourcen befolgt werden, gehört es zum Konzept von Ressourcengruppen, beim Bereitstellen von Ressourcen eine gemeinsame Konfiguration zu verwenden.</span><span class="sxs-lookup"><span data-stu-id="e03e0-136">To ensure that governance policies are applied when resources are created, part of resource grouping design involves using a common configuration when deploying resources.</span></span>

<span data-ttu-id="e03e0-137">Durch die Kombination aus Ressourcengruppen und standardisierten Resource Manager-Vorlagen können Sie Standards dahingehend erzwingen, welche Einstellungen in einer Bereitstellung erforderlich sind und welche [Azure-Richtlinienregeln](/azure/governance/policy/overview) für die einzelnen Ressourcengruppen oder Ressourcen gelten sollen.</span><span class="sxs-lookup"><span data-stu-id="e03e0-137">By combining resource groups and standardized Resource Manager templates, you can enforce standards for what settings are required in a deployment and what [Azure Policy](/azure/governance/policy/overview) rules are applied to each resource group or resource.</span></span>

<span data-ttu-id="e03e0-138">So kann es beispielsweise erforderlich sein, dass alle virtuellen Computer, die in Ihrem Abonnement bereitgestellt werden, mit einem gemeinsamen Subnetz verbunden sind, das von Ihrem zentralen IT-Team verwaltet wird.</span><span class="sxs-lookup"><span data-stu-id="e03e0-138">For example, you may have a requirement that all virtual machines deployed within your subscription connect to a common subnet managed by your central IT team.</span></span> <span data-ttu-id="e03e0-139">Sie können eine Standardvorlage für die Bereitstellung von Workload-VMs erstellen, die eine separate Ressourcengruppe für die Workload erstellt und die erforderlichen VMs in dieser bereitstellt.</span><span class="sxs-lookup"><span data-stu-id="e03e0-139">You can create a standard template for deploying workload VMs which would create a separate resource group for the workload and deploy the required VMs there.</span></span> <span data-ttu-id="e03e0-140">Diese Ressourcengruppe weist eine Richtlinienregel auf, gemäß der nur Netzwerkschnittstellen innerhalb der Ressourcengruppe mit dem gemeinsamen Subnetz verbunden werden dürfen.</span><span class="sxs-lookup"><span data-stu-id="e03e0-140">This resource group would have a policy rule to only allow network interfaces within the resource group to be joined to the shared subnet.</span></span>

<span data-ttu-id="e03e0-141">Eine ausführlichere Erläuterung der Erzwingung Ihrer Richtlinienentscheidungen innerhalb einer Cloudbereitstellung finden Sie unter [Richtlinienerzwingung](../policy-enforcement/overview.md).</span><span class="sxs-lookup"><span data-stu-id="e03e0-141">For a more in-depth discussion of enforcing your policy decisions within a cloud deployment, see [Policy enforcement](../policy-enforcement/overview.md).</span></span>

## <a name="hierarchical-consistency"></a><span data-ttu-id="e03e0-142">Hierarchische Konsistenz</span><span class="sxs-lookup"><span data-stu-id="e03e0-142">Hierarchical consistency</span></span>

<span data-ttu-id="e03e0-143">Wenn die Größe Ihres Cloudumfelds wächst, müssen Sie möglicherweise komplexere Governance-Anforderungen unterstützen, als dies mit der Unternehmens-, Abteilungs-, Konten- und Abonnementhierarchie des Azure Enterprise Agreement möglich ist.</span><span class="sxs-lookup"><span data-stu-id="e03e0-143">As the size of your cloud estate grows, you may need to support more complicated governance requirements than can be supported using the Azure Enterprise Agreement's Enterprise/Department/Account/Subscription hierarchy.</span></span> <span data-ttu-id="e03e0-144">Ressourcengruppen ermöglichen Ihnen, zusätzliche Hierarchieebenen innerhalb Ihrer Organisation zu unterstützen, indem Sie Azure Policy-Regeln und Zugriffssteuerungen auf Ressourcengruppenebene anwenden.</span><span class="sxs-lookup"><span data-stu-id="e03e0-144">Resource groups allows you to support additional levels of hierarchy within your organization, applying Azure Policy rules and access controls at a resource group level.</span></span>

<span data-ttu-id="e03e0-145">[Azure-Verwaltungsgruppen](../subscriptions/overview.md#management-groups) können komplexere Organisationsstrukturen unterstützen, indem Ihre Enterprise Agreement-Struktur mit einer alternativen Hierarchie überlagert wird.</span><span class="sxs-lookup"><span data-stu-id="e03e0-145">[Azure management groups](../subscriptions/overview.md#management-groups) can support more complicated organizational structures by overlaying an alternative hierarchy on top of your enterprise agreement's structure.</span></span> <span data-ttu-id="e03e0-146">Dadurch können Abonnements und die darin enthaltenen Ressourcen Zugriffssteuerungs- und Richtlinienerzwingungsmechanismen unterstützen, die so organisiert sind, dass sie den Anforderungen Ihrer Organisation entsprechen.</span><span class="sxs-lookup"><span data-stu-id="e03e0-146">This allows subscriptions, and the resources they contain, to support access control and policy enforcement mechanisms organized to match your business organizational requirements.</span></span>

## <a name="automated-consistency"></a><span data-ttu-id="e03e0-147">Automatisierte Konsistenz</span><span class="sxs-lookup"><span data-stu-id="e03e0-147">Automated consistency</span></span>
